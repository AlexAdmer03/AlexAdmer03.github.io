Understanding Sorting Algorithms

Key Concepts Explained

Time Complexity

Measures how the algorithm's running time increases with input size:

  • O(n²): If you double the array size, time increases by 4x
  • O(n log n): Much more efficient - doubling array size less than doubles the time

Space Complexity

Measures how much extra memory the algorithm needs:

  • O(1): Uses the same amount of extra memory regardless of array size
  • O(n): Needs extra memory proportional to array size
  • O(log n): Needs extra memory, but less than O(n)

Stability

A sorting algorithm is "stable" if it maintains the relative order of equal elements.

Example with numbers [5, 3, 5, 1] where first 5 is marked as 5a:

  • Before: [5a, 3, 5b, 1]
  • Stable sort: [1, 3, 5a, 5b] - Original order of 5s maintained
  • Unstable sort: Might become [1, 3, 5b, 5a]

Sorting Algorithms Compared

Bubble Sort

Like bubbles rising to the surface, larger elements "bubble up" to their correct positions.

  • Time Complexity: O(n²) - Slow for large arrays
  • Space Complexity: O(1) - Very memory efficient
  • Best for: Small arrays (< 50 elements)
  • Stable: Yes - Maintains order of equal elements
  • Process: Repeatedly compares adjacent elements and swaps them if needed

Quick Sort

Divides and conquers by using a 'pivot' to partition the array into smaller parts.

  • Time Complexity: O(n log n) - Efficient for large arrays
  • Space Complexity: O(log n) - Moderately memory efficient
  • Best for: Large arrays and general-purpose sorting
  • Stable: No - May change order of equal elements
  • Process: Picks a pivot, moves smaller elements left and larger right, then sorts the parts

Merge Sort

Splits array in half, sorts each half, then merges them back together.

  • Time Complexity: O(n log n) - Efficient for large arrays
  • Space Complexity: O(n) - Needs extra memory
  • Best for: Large arrays and when stability is important
  • Stable: Yes - Maintains order of equal elements
  • Process: Divides array until single elements, then merges them in sorted order

Real-World Usage

Different algorithms are used in different scenarios:

  • Bubble Sort: Educational purposes and tiny datasets
  • Quick Sort: Default sort in many programming languages
  • Merge Sort: Databases and when data stability is crucial
